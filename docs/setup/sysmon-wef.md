# Sysmon and Windows Event Forwarding Setup

## Purpose

Sysmon and Windows Event Forwarding provide centralized Windows telemetry for Project Janus. Sysmon records endpoint activity on the Windows 11 client, while WEF forwards those events to Windows Server for review.

## Architecture

```text
Windows 11 Endpoint
    Sysmon 15.2
    Olaf Hartong configuration
            |
            | Source-initiated WEF
            | WinRM over HTTP/5985
            v
Windows Server
    Subscription: Sysmon
    Destination: Forwarded Events
```

## Sysmon Configuration

| Item | Value |
|---|---|
| Endpoint | Windows 11 Enterprise |
| Sysmon version | 15.2 |
| Configuration | Olaf Hartong |
| Modified | No |
| Executable | `Sysmon64.exe` |
| Install command | `Sysmon64.exe -accepteula -i sysmonconfig.xml` |

## WEF Configuration

| Item | Value |
|---|---|
| Subscription name | `Sysmon` |
| Type | Source initiated |
| Destination log | `Forwarded Events` |
| Transport | HTTP |
| Port | `5985` |
| Events forwarded | All Sysmon events |
| Source computer | `purplelab-win-client.purplelab.local` |

## Confirmed Event IDs

- `3` - Network connection
- `11` - File creation
- `13` - Registry value set
- `22` - DNS query

## Test Actions

Telemetry was generated by:

- Opening applications such as Calculator
- Initiating web connections
- Creating files
- Deleting files
- Performing DNS queries
- Logging in and out

## Validation

- `Sysmon64.exe` was visible as a running process
- The local Sysmon Operational log contained events
- The WEF subscription reported the source computer as active
- Forwarded events appeared on Windows Server
- Expanded events preserved process, user, network, registry, DNS, filename, and timestamp fields

## Troubleshooting

Events initially did not arrive on Windows Server.

Troubleshooting included:

1. Deleting and recreating the subscription
2. Verifying WinRM
3. Checking client-to-server connectivity
4. Correcting FQDN-related issues
5. Replacing HTTPS/5986 with HTTP/5985

The HTTP/5985 configuration resolved forwarding.

## Current Workflow Decision

WEF works and was successfully validated. It is not being used as the final log-export workflow because reliable event export was not achieved. Local or script-based export will be used instead for the evidence pipeline.

## Limitations

- No dedicated SIEM
- All Sysmon events are forwarded, which may create unnecessary volume
- Forwarding works, but export reliability remains a problem
- Retention and event filtering have not yet been tuned
